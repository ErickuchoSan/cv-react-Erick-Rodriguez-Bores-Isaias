# Buenas Prácticas para CV de Programador

## 1. Estructura Recomendada

### Orden de Secciones (Prioridad)
1. **Header**: Nombre, título, contacto
2. **Perfil Profesional**: 2-3 oraciones impactantes
3. **Habilidades Técnicas**: Keywords para ATS
4. **Experiencia Laboral**: Más reciente primero
5. **Educación**: Grado y formación continua
6. **Proyectos**: Solo los más relevantes
7. **Idiomas**: Si aplica

## 2. Sección de Experiencia

### Formato STAR para Logros
- **S**ituación: Contexto breve
- **T**area: Qué debías hacer
- **A**cción: Qué hiciste (con tecnologías)
- **R**esultado: Impacto cuantificado

### Verbos de Acción por Categoría

**Desarrollo:**
- Desarrollé, Implementé, Construí, Codifiqué, Programé

**Arquitectura:**
- Diseñé, Arquitecté, Estructuré, Modelé

**Optimización:**
- Optimicé, Mejoré, Refactoricé, Reduje, Aceleré

**Liderazgo:**
- Lideré, Coordiné, Mentoricé, Supervisé

**Colaboración:**
- Colaboré, Integré, Contribuí, Participé

### Cuantificación de Logros

```
❌ "Mejoré el rendimiento de la base de datos"
✅ "Optimicé consultas SQL reduciendo tiempo de respuesta en 40%"

❌ "Desarrollé una API"
✅ "Desarrollé API REST con 6 endpoints, procesando 10K requests/día"

❌ "Automaticé tareas"
✅ "Automaticé procesos con scripts Python, eliminando 8 hrs/semana de trabajo manual"
```

## 3. Sección de Habilidades

### Agrupar por Categoría
```
Backend: C#, .NET Core, Entity Framework, SQL Server
Frontend: React, JavaScript, TypeScript, Tailwind CSS
Cloud: Azure, Azure Functions, Azure Service Bus
DevOps: Git, GitHub, CI/CD, Docker
```

### Incluir Años de Experiencia
```
C# / .NET Core (3 años) | React 19 (2 años) | SQL Server (3 años)
```

### Priorizar según Job Description
Poner primero las tecnologías que pide el trabajo.

## 4. Formato ATS-Friendly

### Keywords Esenciales para .NET/React

**Backend:**
- C#, .NET, .NET Core, ASP.NET, Entity Framework
- SQL Server, T-SQL, PostgreSQL
- REST API, Web API, Microservices
- SOLID, Clean Architecture, DDD, Design Patterns

**Frontend:**
- React, JavaScript, TypeScript, Node.js
- HTML5, CSS3, Tailwind, Bootstrap
- Redux, Context API, Hooks

**Cloud/DevOps:**
- Azure, AWS, Docker, Kubernetes
- CI/CD, GitHub Actions, Azure DevOps

**Seguridad:**
- OAuth2, JWT, Authentication, Encryption

## 5. Errores a Evitar

### Contenido
- ❌ Listar tecnologías sin contexto de uso
- ❌ Logros sin cuantificar
- ❌ Descripción genérica de responsabilidades
- ❌ Incluir tecnologías que no dominas
- ❌ Gaps de empleo sin explicación

### Formato
- ❌ Más de 2 páginas
- ❌ Fuentes decorativas
- ❌ Colores excesivos
- ❌ Múltiples columnas (rompe ATS)
- ❌ Gráficos/iconos en PDF ATS

### Técnico
- ❌ Email no profesional
- ❌ Links rotos
- ❌ Información desactualizada
- ❌ Inconsistencias entre versiones

## 6. Personalización por Trabajo

### Antes de Aplicar
1. Leer job description completo
2. Identificar keywords técnicos
3. Verificar que aparecen en tu CV
4. Ajustar orden de skills si necesario
5. Verificar que experiencia es relevante

### Secciones a Personalizar
- Perfil profesional (enfatizar lo que piden)
- Orden de habilidades (poner primero lo que piden)
- Descripción de logros (resaltar los más relevantes)

## 7. Métricas de un Buen CV

### Longitud
- **Junior (0-2 años)**: 1 página
- **Mid (2-5 años)**: 1-2 páginas
- **Senior (5+ años)**: 2 páginas máximo

### Ratio Contenido
- 60% Experiencia y logros
- 20% Habilidades técnicas
- 15% Educación y proyectos
- 5% Contacto y header

### Test de 6 Segundos
El reclutador debe entender en 6 segundos:
- ✅ Qué rol buscas
- ✅ Años de experiencia
- ✅ Stack tecnológico principal
- ✅ Logro más impactante
